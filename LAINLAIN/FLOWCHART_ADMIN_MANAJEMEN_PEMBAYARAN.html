Penjelasan Flowchart Admin - Manajemen Pembayaran

Flowchart ini menggambarkan alur manajemen pembayaran manual oleh admin untuk pembelian kuota oleh user. Proses dimulai ketika admin mengakses menu manajemen billing melalui dashboard admin. Admin kemudian melihat daftar semua pembayaran yang memiliki status pending, yaitu pembayaran yang telah diupload oleh user tetapi belum diproses oleh admin. Setelah melihat daftar, admin dapat memilih untuk menyetujui atau menolak pembayaran tersebut.

Jika admin memilih untuk approve pembayaran, sistem akan mengubah status purchase menjadi approved dan menambahkan kuota sesuai jumlah yang ada di package ke total_quota user. Jika admin memilih untuk reject pembayaran, sistem hanya akan mengubah status purchase menjadi rejected tanpa menambahkan kuota ke user, sehingga pembayaran ditolak tetapi user tidak mendapatkan kuota yang dibayarkan. Setelah operasi selesai, admin dapat melihat hasilnya di dashboard.
