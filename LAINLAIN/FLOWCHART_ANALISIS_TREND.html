📋 Penjelasan Flowchart: Algoritma Analisis Trend Ranking

1. Overview

Flowchart ini menggambarkan algoritma perhitungan trend dan stability dari data ranking history. Algoritma ini menganalisis perubahan ranking dari waktu ke waktu untuk menentukan apakah ranking meningkat, menurun, atau stabil, serta menentukan apakah pergerakan ranking stabil atau fluktuatif.

2. Alur Proses

A. Inisialisasi & Validasi Data

  • Input: rank_history (array of objects dengan {date, rank, url})
  • Inisialisasi default: trend = 'tidak cukup data', stability = 'tidak cukup data', avg_rank = 0
  • Cek apakah len(rank_history) >= 2 (minimal 2 data untuk analisis)
  • Jika tidak cukup data → Set avg_rank = rank_history[0]['rank'], return dengan default values

B. Perhitungan Trend

  • first_rank: Ranking pertama (index 0) - ranking tertua
  • last_rank: Ranking terakhir (index -1) - ranking terbaru
  • rank_change: first_rank - last_rank
  • Decision logic:
    • Jika rank_change > 0 → trend = 'meningkat' (ranking naik, angka lebih kecil = lebih baik)
    • Jika rank_change < 0 → trend = 'menurun' (ranking turun, angka lebih besar = lebih buruk)
    • Jika rank_change == 0 → trend = 'stabil' (ranking tidak berubah)

C. Perhitungan Average Rank

  • avg_rank = sum(rank_history[i]['rank']) / len(rank_history)
  • Menghitung rata-rata ranking dari semua data history

D. Perhitungan Variance & Stability (LOOP)

  • Inisialisasi rank_variance = 0
  • Loop melalui semua rank_history:
    • Untuk setiap rank_history[i]:
    • diff = rank_history[i]['rank'] - avg_rank
    • square = diff² (variance calculation)
    • rank_variance += square
  • Setelah loop: rank_variance = rank_variance / len(rank_history)
  • Decision logic:
    • Jika rank_variance < 5 → stability = 'stabil' (perubahan kecil)
    • Jika rank_variance >= 5 → stability = 'fluktuatif' (perubahan besar)

E. Return Result

  • Return object: {trend, stability, avg_rank}
  • Data ini digunakan sebagai metadata untuk SEO Analysis

3. Decision Points

  • 5 Decision Points: Validasi data, trend direction (2 checks), loop iteration, stability threshold
  • Loop: Iterasi melalui semua rank_history untuk menghitung variance

4. Formula Matematis

  • Average Rank: μ = (Σ rank_i) / n
  • Variance: σ² = Σ(rank_i - μ)² / n
  • Threshold Stability: variance < 5 = stabil, >= 5 = fluktuatif

5. Interpretasi Hasil

  • Trend 'meningkat': Ranking naik (posisi lebih baik), angka rank lebih kecil
  • Trend 'menurun': Ranking turun (posisi lebih buruk), angka rank lebih besar
  • Trend 'stabil': Ranking tidak berubah
  • Stability 'stabil': Perubahan ranking konsisten, variance rendah
  • Stability 'fluktuatif': Perubahan ranking tidak konsisten, variance tinggi

6. Kompleksitas

  • Time Complexity: O(n) dimana n = jumlah data rank_history
  • Space Complexity: O(1) - hanya menggunakan variabel konstan

Penjelasan Flowchart Algoritma Analisis Trend Ranking

Flowchart ini menggambarkan algoritma analisis trend dan stabilitas ranking yang digunakan untuk menganalisis pergerakan posisi ranking website dari waktu ke waktu. Algoritma ini memerlukan minimal 2 data history ranking untuk dapat melakukan analisis yang akurat. Proses dimulai dengan validasi jumlah data history, kemudian sistem akan menghitung trend ranking dengan membandingkan ranking pertama (ranking tertua) dengan ranking terakhir (ranking terbaru). Perhitungan trend didasarkan pada perubahan ranking, dimana jika ranking naik (angka lebih kecil berarti posisi lebih baik), trend dinyatakan sebagai 'meningkat', sebaliknya jika ranking turun dinyatakan sebagai 'menurun', dan jika tidak ada perubahan dinyatakan sebagai 'stabil'.

Sistem juga menghitung average rank dengan menjumlahkan semua ranking dan membaginya dengan jumlah data history. Untuk menentukan stabilitas ranking, sistem menggunakan perhitungan variance yang dilakukan melalui iterasi terhadap semua data history dengan rumus statistik standar. Jika variance kurang dari 5, stabilitas dinyatakan sebagai 'stabil' yang menunjukkan perubahan ranking yang konsisten, sedangkan jika variance 5 atau lebih dinyatakan sebagai 'fluktuatif' yang menunjukkan perubahan ranking yang tidak konsisten. Hasil analisis trend dan stabilitas ini kemudian digunakan sebagai metadata dalam proses SEO Analysis dengan AI untuk memberikan konteks yang lebih kaya dalam analisis.
