📋 Penjelasan Flowchart: Algoritma Validasi & Pengurangan Kuota

1. Overview

Flowchart ini menggambarkan algoritma validasi dan pengurangan kuota user. Algoritma ini digunakan sebelum melakukan aksi yang membutuhkan kuota (seperti cek rank atau SEO analysis) untuk memastikan user memiliki kuota yang cukup.

2. Alur Proses

A. Input Parameter

  • user: User object yang akan divalidasi
  • action_type: Tipe aksi yang akan dilakukan (opsional, untuk logging)
  • quota_cost: Jumlah kuota yang dibutuhkan untuk aksi tersebut

B. Validasi UserStatus

  • Cek apakah UserStatus exists untuk user tersebut
  • Jika tidak ada → Error: UserStatus tidak ditemukan, proses berakhir
  • Jika ada → Ambil UserStatus object dari database

C. Perhitungan Remaining Quota

  • Hitung remaining quota: remaining = total_quota - used_quota
  • Formula ini menggunakan method remaining_quota() dari model UserStatus

D. Validasi Kuota Cukup

  • Cek apakah remaining >= quota_cost
  • Jika tidak cukup → Error dengan informasi sisa kuota dan kuota yang dibutuhkan
  • Jika cukup → Lanjut ke pengurangan kuota

E. Pengurangan Kuota

  • Update used_quota: user_status.used_quota += quota_cost
  • Save perubahan ke database: user_status.save()
  • Return success message

3. Decision Points

  • 2 Decision Points: Validasi UserStatus exists, validasi kuota cukup
  • Error Handling: Setiap validasi yang gagal akan mengembalikan error message spesifik

4. Penggunaan di Sistem

  • Cek Rank: quota_cost = 1 (setiap cek rank mengurangi 1 kuota)
  • SEO Analysis: quota_cost bervariasi berdasarkan model AI:
    • Gemini: 3 kuota
    • GPT-3.5: 2 kuota
    • GPT-4.1: 5 kuota

5. Keamanan & Konsistensi

  • Atomic Operation: Validasi dan update dilakukan dalam satu flow untuk mencegah race condition
  • Database Transaction: Save operation memastikan data konsisten
  • Error Prevention: Validasi dilakukan sebelum aksi untuk mencegah aksi dengan kuota tidak cukup

6. Catatan Implementasi

  • Algoritma ini diimplementasikan secara inline di views.py, bukan sebagai fungsi terpisah
  • Validasi kuota dilakukan di awal proses (sebelum request API) untuk efisiensi
  • Pengurangan kuota dilakukan setelah proses sukses (setelah mendapatkan hasil) untuk akurasi

Penjelasan Flowchart Algoritma Validasi & Pengurangan Kuota

Flowchart ini menggambarkan algoritma validasi dan pengurangan kuota yang merupakan komponen penting dalam sistem manajemen kuota. Algoritma ini digunakan di berbagai proses dalam sistem, seperti pengecekan ranking dan analisis SEO dengan AI. Proses dimulai dengan validasi keberadaan UserStatus untuk memastikan pengguna memiliki record status di database, kemudian sistem akan menghitung sisa kuota yang tersedia dengan mengurangi total kuota dengan kuota yang telah digunakan.

Sistem kemudian membandingkan sisa kuota dengan biaya kuota yang diperlukan untuk aksi yang akan dilakukan, dimana biaya kuota bervariasi tergantung jenis aksi. Jika sisa kuota mencukupi, sistem akan melakukan pengurangan kuota dengan menambahkan biaya kuota ke used_quota dan menyimpan perubahan ke database. Operasi ini dirancang secara atomic untuk mencegah race condition dan memastikan konsistensi data. Validasi kuota dilakukan di awal proses sebelum melakukan request ke API eksternal untuk efisiensi, sedangkan pengurangan kuota dilakukan setelah proses sukses untuk memastikan kuota hanya dikurangi ketika hasil berhasil diperoleh.
