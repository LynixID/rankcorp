📋 Penjelasan Activity Diagram: Admin - Manajemen User

1. Overview

Activity Diagram ini menggambarkan alur proses manajemen user oleh admin. Admin dapat melakukan operasi CRUD (Create, Read, Update, Delete) dan toggle status aktif/nonaktif user. Proses ini melibatkan dua aktor: Admin dan System.

2. Alur Proses

A. Akses Dashboard

  • Admin akses dashboard admin dan melihat daftar semua user
  • Admin dapat memilih aksi: Update, Delete, atau Toggle Status

B. Update User

  • Admin klik edit user → Update data (username, email, WhatsApp, role)
  • System update CustomUser → Tampilkan pesan sukses

C. Delete User

  • Admin klik delete → Konfirmasi hapus
  • Jika konfirmasi: System delete CustomUser (CASCADE ke UserStatus, RankResult, dll)
  • System tampilkan pesan sukses

D. Toggle Status

  • Admin klik toggle status → System set is_active (True/False)
  • System tampilkan pesan sukses

3. Fitur Keamanan

  • Hanya admin yang dapat mengakses fitur ini
  • Konfirmasi diperlukan untuk delete user
  • CASCADE delete memastikan data terkait juga terhapus
