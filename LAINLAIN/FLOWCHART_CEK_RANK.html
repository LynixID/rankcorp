📋 Penjelasan Flowchart: Algoritma Cek Rank & Pencarian Domain

1. Overview

Flowchart ini menggambarkan algoritma pencarian domain di hasil pencarian Google menggunakan SerpAPI. Algoritma ini melakukan iterasi melalui hasil pencarian organik dan mencocokkan domain yang dicari dengan link yang ditemukan.

2. Alur Proses

A. Validasi Awal

  • Cek UserStatus: Pastikan user memiliki UserStatus di database
  • Cek Kuota: Pastikan remaining_quota > 0 (kuota tersedia)
  • Jika salah satu gagal → Error message, proses berakhir

B. Konfigurasi API

  • Cek apakah ApiConfig ada di database (api_name='serpapi', is_active=True)
  • Jika ada → Ambil api_key, base_url, timeout dari database
  • Jika tidak ada → Gunakan fallback hardcoded (untuk kompatibilitas)

C. Request ke SerpAPI

  • Build parameter request: engine='google', q=keyword, google_domain, hl, gl, num, api_key
  • Request GET ke SerpAPI dengan timeout yang ditentukan
  • Jika request gagal → Error message, proses berakhir

D. Parsing & Iterasi (LOOP)

  • Parse JSON response dari SerpAPI
  • Inisialisasi array ranks = []
  • Cek apakah "organic_results" ada di response
  • Jika tidak ada → Return ranks kosong
  • Jika ada → Mulai loop iterasi:
    • Untuk setiap result di organic_results:
    • Ambil link dari result
    • Cek apakah domain ada di link (pattern matching)
    • Jika cocok → Append {position: idx+1, link} ke ranks
    • Simpan RankResult ke database (user, domain, keyword, rank, url_result)
    • Lanjut ke result berikutnya

E. Update Kuota & Return

  • Setelah loop selesai → Update used_quota += 1
  • Build result_data dengan informasi: domain, keyword, num, total_found, ranks
  • Return result_data untuk ditampilkan ke user

3. Decision Points

  • 7 Decision Points: UserStatus, kuota, konfigurasi API, response, organic_results, domain matching, iterasi loop
  • Loop: Iterasi melalui semua organic_results untuk mencari domain yang cocok
  • Pattern Matching: Menggunakan operator 'in' untuk mencari domain di link (substring matching)

4. Fitur Penting

  • Multiple Rank Detection: Bisa menemukan beberapa posisi jika domain muncul di beberapa hasil
  • Database Storage: Setiap rank yang ditemukan disimpan ke RankResult untuk history
  • Quota Management: Kuota dikurangi setelah proses selesai (bukan sebelum request)
  • Error Handling: Fallback ke hardcoded config jika database config tidak tersedia

5. Kompleksitas

  • Time Complexity: O(n) dimana n = jumlah organic_results
  • Space Complexity: O(m) dimana m = jumlah rank yang ditemukan

Penjelasan Flowchart Algoritma Cek Rank & Pencarian Domain

Flowchart ini menggambarkan algoritma utama dalam sistem rank checker untuk melakukan pengecekan posisi ranking website di hasil pencarian Google. Proses dimulai dengan validasi kuota pengguna untuk memastikan pengguna memiliki kuota yang cukup sebelum melakukan request ke API eksternal. Setelah validasi berhasil, sistem akan melakukan request ke SerpAPI dengan parameter pencarian yang telah ditentukan oleh pengguna, seperti keyword, domain target, dan parameter lokasi pencarian.

Setelah menerima response dari SerpAPI, sistem akan melakukan parsing terhadap data JSON dan melakukan iterasi melalui semua hasil pencarian organik menggunakan teknik pattern matching untuk mencari domain yang dicari di dalam setiap link. Jika domain ditemukan, posisi ranking akan disimpan ke dalam database sebagai history dan ditampilkan kepada pengguna. Algoritma ini dirancang dengan kompleksitas waktu O(n) dan pengurangan kuota dilakukan setelah proses selesai untuk memastikan kuota hanya dikurangi ketika proses berhasil mendapatkan hasil. Sistem juga dilengkapi dengan error handling yang komprehensif untuk menangani berbagai skenario kegagalan.
