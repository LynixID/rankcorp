📋 Penjelasan Flowchart: Proses SEO Analysis dengan AI

1. Overview

Flowchart ini menggambarkan algoritma proses analisis SEO menggunakan AI (Gemini atau OpenAI). Proses ini hanya tersedia untuk user premium dan melibatkan routing ke API yang berbeda berdasarkan model yang dipilih.

2. Alur Proses

A. Validasi Awal (5 tahap)

  • Validasi UserStatus: Pastikan user memiliki UserStatus
  • Validasi Premium: Pastikan status == 'premium' (fitur premium only)
  • Validasi Input: Pastikan keyword dan domain tidak kosong
  • Validasi Model: Pastikan model_choice valid (gemini, gpt-3.5-turbo, gpt-4-1106-preview)
  • Validasi Kuota: Pastikan remaining_quota >= quota_cost
  • Jika salah satu gagal → Error message spesifik, proses berakhir

B. Perhitungan Trend & Build Prompt

  • Hitung trend dan stability menggunakan algoritma analisis trend (lihat Flowchart #3)
  • Build prompt dengan format yang mencakup:
    • Data website (keyword, domain, jumlah data history)
    • Histori ranking lengkap
    • Instruksi analisis (ringkasan, analisis tren, penyebab, rekomendasi, target)

C. Routing Model AI

  • Decision: model_choice == 'gemini'?
  • Jika Ya → Masuk ke GEMINI FLOW
  • Jika Tidak → Masuk ke OPENAI FLOW

D. GEMINI FLOW

  • Ambil API_KEY dari model_config.api_key
  • Validasi API_KEY ada → Jika tidak, error
  • Build payload:
    • URL: Gemini API endpoint (gemini-2.0-flash:generateContent)
    • Headers: X-goog-api-key, Content-Type
    • Payload: contents (dengan prompt), generationConfig (temperature=0.7, maxOutputTokens=2048)
  • POST request ke Gemini API
  • Parse response: candidates[0].content.parts[0].text

E. OPENAI FLOW

  • Decision: model_choice == 'gpt-3.5-turbo'?
  • Jika Ya → model = 'gpt-3.5-turbo', max_tokens = 2500
  • Jika Tidak → model = 'gpt-4-1106-preview', max_tokens = 3000
  • Build payload:
    • URL: OpenAI API endpoint (chat/completions)
    • Headers: Authorization (Bearer API_KEY), Content-Type
    • Payload: model, messages (system + user), max_tokens, temperature=0.7
  • POST request ke OpenAI API
  • Parse response: choices[0].message.content

F. Validasi & Penyimpanan

  • Cek response status == 200 → Jika tidak, error
  • Cek analysis length > 50 → Jika tidak, error (hasil tidak memadai)
  • Simpan SeoAnalysis ke database dengan metadata (trend, stability, avg_rank, model_used)
  • Update kuota: used_quota += quota_cost
  • Return result dengan analysis dan metadata

3. Decision Points

  • 11 Decision Points: UserStatus, premium, input, model, kuota, routing model, Gemini API key, OpenAI model selection, response status (2x), analysis length
  • Branching: Routing ke 2 flow berbeda (Gemini vs OpenAI) berdasarkan model choice

4. Quota Cost per Model

  • Gemini: 3 kuota (cepat & murah)
  • GPT-3.5: 2 kuota (stabil, harga menengah)
  • GPT-4.1: 5 kuota (terbaik, analisis mendalam)

5. Error Handling

  • API Errors: Handle status code 400, 401, 402, 429, 500 dengan message spesifik
  • Timeout: Handle request timeout (60 detik)
  • Connection Error: Handle koneksi gagal
  • Validation Errors: Setiap validasi memiliki error message spesifik

6. Fitur Premium

  • Hanya user dengan status 'premium' yang bisa mengakses fitur ini
  • User 'guest' akan mendapat error dengan pesan untuk upgrade ke premium
  • Status premium didapat setelah melakukan pembelian kuota pertama kali

Penjelasan Flowchart Proses SEO Analysis dengan AI

Flowchart ini menggambarkan proses analisis SEO menggunakan teknologi Artificial Intelligence yang merupakan fitur eksklusif untuk pengguna premium. Proses dimulai dengan validasi status pengguna untuk memastikan hanya pengguna dengan status 'premium' yang dapat mengakses fitur ini. Sistem kemudian melakukan validasi terhadap data input yang mencakup keyword, domain target, history ranking, dan pilihan model AI yang akan digunakan. Pengguna dapat memilih antara tiga model AI yang tersedia, yaitu Gemini dari Google, GPT-3.5, atau GPT-4.1 dari OpenAI, dengan biaya kuota yang berbeda untuk setiap model.

Setelah validasi berhasil, sistem akan menghitung trend dan stabilitas ranking berdasarkan data history yang kemudian digunakan untuk membangun prompt yang komprehensif untuk AI. Sistem akan melakukan routing berdasarkan model AI yang dipilih, dengan alur yang berbeda untuk Gemini dan OpenAI. Setelah menerima response dari API AI, sistem akan melakukan validasi terhadap kualitas hasil analisis dan menyimpan hasil yang valid ke database beserta metadata seperti trend, stability, average rank, dan model yang digunakan. Sistem juga akan mengurangi kuota pengguna sesuai dengan biaya yang ditetapkan, dan proses ini dilengkapi dengan error handling yang komprehensif untuk menangani berbagai skenario kegagalan.
