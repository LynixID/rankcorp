Penjelasan Flowchart Admin - Manajemen API Config

Flowchart ini menggambarkan alur manajemen konfigurasi API oleh admin dalam sistem rank checker, khususnya untuk mengelola konfigurasi SerpAPI dan API pencarian lainnya. Proses dimulai ketika admin mengakses menu manajemen API config melalui dashboard admin. Admin kemudian melihat daftar semua konfigurasi API yang tersedia, termasuk informasi seperti API name, base URL, max results, timeout, dan status aktif. Setelah melihat daftar, admin dapat memilih untuk mengedit konfigurasi API atau membuka debug popup untuk melihat detail konfigurasi dan melakukan test API.

Jika admin memilih untuk edit, admin akan mengisi form update konfigurasi API seperti API key, base URL, max results, dan timeout, kemudian sistem akan menyimpan data konfigurasi API tersebut. Setelah data disimpan, admin akan kembali ke daftar API configs. Jika admin memilih untuk debug, sistem akan membuka popup yang menampilkan informasi lengkap konfigurasi API termasuk API name, API key, base URL, max results, timeout, description, status, dan created at dalam format yang mudah dibaca. Setelah popup muncul, sistem secara otomatis menjalankan test API dengan parameter yang sudah ditentukan: keyword "serp api", domain "serpapi.com", dan region Indonesia.

Sistem menggunakan algoritma yang sama dengan fitur search rank untuk mencari domain tersebut di hasil pencarian. Setelah proses test selesai, sistem akan memproses hasil test. Jika berhasil, sistem akan menampilkan hasil rank (posisi domain di hasil pencarian) dan mengirimkan notifikasi berhasil. Jika gagal, sistem akan menampilkan pesan error yang menjelaskan penyebab kegagalan. Setelah melihat hasil test, admin akan kembali ke daftar API configs.
