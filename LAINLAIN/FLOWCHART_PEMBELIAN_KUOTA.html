📋 Penjelasan Flowchart: Proses Pembelian Kuota (Manual Transfer)

1. Overview

Flowchart ini menggambarkan algoritma proses pembelian kuota dengan metode manual transfer. Proses ini melibatkan checkout session management dengan expiry time, upload bukti pembayaran, dan approval workflow oleh admin.

2. Alur Proses

A. Checkout Session Management

  • User memilih package (package_id)
  • Cek apakah user memiliki active session yang belum expired
  • Jika ada dan tidak expired → Redirect langsung ke upload page
  • Jika expired → Update session status menjadi 'expired'
  • Cek existing session untuk package yang sama
  • Jika tidak ada atau expired → Buat session baru:
    • Generate session_id menggunakan UUID
    • Set expires_at = now + 2 hours (session berlaku 2 jam)
    • Buat CheckoutSession dengan status='active'

B. Upload Payment Proof

  • User diarahkan ke upload page
  • Validasi session masih aktif dan tidak expired
  • Jika session tidak valid → Error, redirect ke buy quota page
  • User upload file bukti pembayaran
  • Validasi form (file harus ada, format valid)
  • Jika form tidak valid → Error, user bisa upload ulang
  • Jika valid → Buat QuotaPurchase:
    • user, package, payment_proof (file), status='pending'
  • Update CheckoutSession status menjadi 'completed'
  • Redirect ke purchase status page (menunggu approval)

C. Admin Approval Workflow

  • Admin melihat daftar QuotaPurchase dengan status='pending'
  • Admin bisa memilih Approve atau Reject

D. APPROVE FLOW

  • Update QuotaPurchase status menjadi 'approved'
  • Cek apakah UserStatus exists untuk user tersebut
  • Jika tidak ada → Buat UserStatus baru dengan status='premium'
  • Jika ada → Update UserStatus:
    • total_quota += package.quota_amount
    • Update status = 'premium' (jika belum premium)
  • Save perubahan ke database
  • Success: Pembayaran disetujui, kuota ditambahkan

E. REJECT FLOW

  • Update QuotaPurchase status menjadi 'rejected'
  • Kuota tidak ditambahkan
  • User bisa upload bukti baru atau hubungi admin

3. Decision Points

  • 8 Decision Points: Active session, expired check (2x), existing session, session active, form valid, admin action, UserStatus exists
  • State Management: CheckoutSession memiliki status: 'active', 'expired', 'completed', 'cancelled'

4. Session Expiry

  • Durasi: 2 jam dari waktu pembuatan session
  • Validasi: Setiap akses upload page akan cek apakah session expired
  • Auto-expire: Session yang expired akan di-mark sebagai 'expired'
  • Purpose: Mencegah session lama yang tidak digunakan, memastikan user upload bukti dalam waktu wajar

5. Status Upgrade

  • Setelah approval pertama kali, user status otomatis menjadi 'premium'
  • Status 'premium' memberikan akses ke fitur SEO Analysis
  • User 'guest' hanya bisa cek rank, tidak bisa SEO Analysis

6. Error Handling

  • Session Expired: User harus memilih package lagi
  • Form Invalid: User bisa upload ulang bukti
  • Session Not Found: Redirect ke buy quota page

7. Keamanan

  • Session hanya valid untuk user yang membuatnya
  • File upload divalidasi untuk mencegah file berbahaya
  • Admin approval mencegah penipuan dan memastikan pembayaran valid

Penjelasan Flowchart Proses Pembelian Kuota (Manual Transfer)

Flowchart ini menggambarkan proses pembelian kuota melalui metode manual transfer yang memerlukan verifikasi dari administrator. Proses dimulai dengan pemilihan paket kuota yang diinginkan oleh pengguna, kemudian sistem akan memeriksa apakah terdapat session aktif yang masih valid. Jika session tidak ada atau sudah expired, sistem akan membuat session baru dengan durasi validitas 2 jam untuk melacak proses pembelian dan memastikan pengguna mengupload bukti pembayaran dalam waktu yang wajar.

Setelah session terkonfirmasi, pengguna akan diarahkan ke halaman upload bukti pembayaran dan sistem akan melakukan validasi terhadap session sebelum memproses upload file. Jika file valid, sistem akan membuat record QuotaPurchase dengan status 'pending' dan mengupdate status CheckoutSession menjadi 'completed'. Administrator kemudian dapat melihat daftar pembelian yang menunggu persetujuan dan memilih untuk menyetujui atau menolak pembayaran. Jika disetujui, sistem akan menambahkan kuota sesuai paket yang dibeli ke UserStatus pengguna, dan setelah approval pertama kali, status pengguna otomatis menjadi 'premium' yang memberikan akses ke fitur SEO Analysis. Proses ini memastikan keamanan transaksi dan mencegah penipuan melalui verifikasi manual oleh administrator.
