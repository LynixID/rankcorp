Penjelasan Diagram Use Case Sistem Rank Checker

Diagram Use Case ini menggambarkan interaksi antara berbagai aktor dengan sistem rank checker melalui use case yang tersedia. Sistem memiliki empat aktor utama yaitu Guest (pengunjung belum login), User (pengguna terdaftar dengan status guest atau premium), Admin (administrator sistem), dan System (sistem eksternal seperti API). Guest dapat melakukan tiga use case dasar yaitu melihat landing page, melakukan registrasi dengan verifikasi OTP, dan login ke sistem. User yang sudah terdaftar dapat melakukan login/logout, mengakses dashboard, melakukan pengecekan ranking website dengan history dan export PDF, melakukan analisis SEO dengan AI yang merupakan fitur eksklusif untuk user premium, serta membeli kuota melalui metode manual transfer atau payment gateway Midtrans.

Admin memiliki akses untuk mengelola seluruh aspek sistem melalui enam use case yaitu dashboard admin dengan statistik dan grafik, manajemen user dengan operasi CRUD dan toggle status, manajemen pembayaran untuk approve atau reject pembayaran manual, manajemen paket kuota, manajemen konfigurasi AI model, dan manajemen konfigurasi API. System sebagai aktor eksternal bertanggung jawab untuk notifikasi dan callback dari berbagai layanan seperti OTP, payment gateway, dan integrasi dengan API eksternal seperti SerpAPI untuk pencarian Google serta API AI seperti Gemini dan OpenAI untuk analisis SEO. Diagram ini juga menunjukkan relasi include dan extend antar use case yang menggambarkan dependensi dan ekstensi fungsionalitas dalam sistem.
