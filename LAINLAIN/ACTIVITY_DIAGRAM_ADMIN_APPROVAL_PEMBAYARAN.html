📋 Penjelasan Activity Diagram: Admin - Approval Pembayaran

1. Overview

Activity Diagram ini menggambarkan alur proses approval pembayaran manual transfer oleh admin. Admin melakukan review bukti pembayaran dan memutuskan untuk approve atau reject. Proses ini melibatkan dua aktor: Admin dan System.

2. Alur Proses

A. Review Pembayaran

  • Admin akses dashboard dan melihat daftar pembayaran dengan status "Pending"
  • Admin pilih QuotaPurchase dan review bukti pembayaran
  • Admin memutuskan: Valid atau Tidak Valid

B. Approve Pembayaran

  • Jika valid: Admin klik "Approve"
  • System update QuotaPurchase status menjadi "Approved"
  • System update UserStatus: total_quota += package.quota_amount
  • System update UserStatus status menjadi "premium"
  • System tampilkan pesan sukses

C. Reject Pembayaran

  • Jika tidak valid: Admin klik "Reject"
  • System update QuotaPurchase status menjadi "Rejected"
  • System tampilkan pesan sukses

3. Dampak Approval

  • Approve: User mendapatkan kuota dan status premium
  • Reject: User tidak mendapatkan kuota, dapat upload bukti baru
