📋 Penjelasan Activity Diagram: Admin - Manajemen Konfigurasi

1. Overview

Activity Diagram ini menggambarkan alur proses manajemen konfigurasi sistem oleh admin. Admin dapat mengelola konfigurasi AI Model dan API Config. Proses ini melibatkan dua aktor: Admin dan System.

2. Alur Proses - AI Model Config

A. CRUD Operations

  • Add: Admin isi form model baru → System create AiModelConfig
  • Update: Admin edit model → System update AiModelConfig
  • Delete: Admin delete model → System delete AiModelConfig
  • Toggle: Admin toggle is_active → System update status

B. Data yang Dikelola

  • Model Name (gemini, gpt-3.5-turbo, gpt-4-1106-preview)
  • Display Name
  • Quota Cost (biaya kuota per penggunaan)
  • API Key (untuk akses API)
  • Is Active (status aktif/nonaktif)

3. Alur Proses - API Config

A. Update Konfigurasi

  • Admin pilih API Config (SerpAPI, Google Search, Bing)
  • Admin update: API Key, Base URL, Max Results, Timeout
  • System update ApiConfig → Tampilkan pesan sukses

B. Toggle Status

  • Admin toggle is_active → System update status
  • API yang nonaktif tidak akan digunakan oleh system

4. Manfaat

  • Fleksibilitas: Update konfigurasi tanpa deploy ulang
  • Keamanan: API keys disimpan di database
  • Kontrol: Admin dapat mengaktifkan/nonaktifkan model/API
